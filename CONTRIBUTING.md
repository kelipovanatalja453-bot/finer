# 贡献指南 | Contributing to finer

感谢你对 `finer` 的关注！我们欢迎各类型的贡献。
Thank you for your interest in `finer`! We welcome all types of contributions.

## 如何贡献 | How to Contribute

1. **Fork** 仓库并创建你的特性分支。
   **Fork** the repository and create your feature branch.
2. **提交 Issue**：发现 Bug 或有新想法？请先提交 Issue 讨论。
   **Open an Issue**: Found a bug or have an idea? Please open an issue first.
3. **提交 PR**：确保你的代码符合规范并包含必要的测试。
   **Submit a PR**: Ensure your code follows the standards and includes necessary tests.

## 代码规范 | Code Standards

- 使用 **Black** 进行代码格式化。
  Use **Black** for code formatting.
- 确保所有 Pydantic 模型在 `src/finer/schemas/` 中定义。
  Ensure all Pydantic models are defined in `src/finer/schemas/`.
- 提交说明请遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范。
  Follow [Conventional Commits](https://www.conventionalcommits.org/) for your commit messages.
