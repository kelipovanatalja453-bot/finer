# Finer OS UX 信息架构与 KOL 评价系统设计规范

> 版本：v1.0 | 日期：2026-04-23 | 状态：设计规划阶段

---

## 一、现有 Dashboard UX 诊断

### 问题定位

**核心问题**：导航层级过于技术化，用户心智模型与系统架构不匹配；KOL 评价维度缺失，无法支撑观点验证闭环。

### 现状分析

#### 1. 导航层级问题

**当前状态**：Sidebar 展示 L0-L8 共 7 个层级，按技术流水线命名

```
L0 Intake      → 原始素材入库
L1 Enrichment  → 话题拆分与标的归类
L2 Library     → 标准化知识资产池
L3 Parsing     → OCR、视觉转录
L5 Extraction  → 候选事件与意图链
L6 Review      → 字段修正与歧义裁决
L8 Backtest    → 结果验证与收益复盘
```

**痛点识别**：
- 认知负担重：用户需理解 L0-L8 技术含义才能找到目标
- 心智模型错位：用户关心的是「任务」而非「流水线层级」
- 层级过深：7 个平级选项，选择压力过大
- 缺少用户视角入口：KOL、标的、时间维度均无直接入口

#### 2. 信息密度问题

**当前状态**：所有层级使用统一的卡片网格/列表视图

**痛点识别**：
- L0-L2 内容为「文件」，L5-L8 内容为「事件」，但展示方式一致
- 卡片信息过载：图标 + 名称 + 日期 + 大小 + 状态 + 时间戳
- 关键信息被淹没：观点方向、置信度、验证状态等核心信息不突出

#### 3. 操作流程问题

**当前流程**（以观点验证为例）：
```
导入文件(L0) → 查看富化结果(L1) → 归档(L2) → 解析(L3)
→ 抽取事件(L5) → 人工复核(L6) → 回测验证(L8)
```

**痛点识别**：
- 流程割裂：每一步都需要手动切换层级
- 缺少快捷路径：无法从 L0 直达 L5
- 无进度感知：用户不知道资产处于哪个阶段

#### 4. KOL 评价维度缺失

**当前状态**：无 KOL 维度的视图和评价体系

**痛点识别**：
- 无法按 KOL 聚合观点
- 无法追踪 KOL 历史表现
- 无法验证观点准确率
- 缺少评价可视化（星级、时间线）

### 根因分析

1. **技术驱动设计**：UI 结构直接映射后端流水线，而非用户任务
2. **缺少用户画像**：未区分「研究者」「KOL 追踪者」「资产管理者」等角色
3. **信息架构扁平**：所有功能平铺，缺少分层抽象
4. **评价闭环未形成**：有数据无评价，有观点无验证

## 二、信息架构重新设计

### 设计原则

1. **任务导向**：按用户任务组织，而非技术层级
2. **分层导航**：顶层任务分类，底层具体功能
3. **快捷入口**：高频任务直达，减少跳转
4. **状态感知**：资产流转状态始终可见

### 新信息架构

```
Finer OS
├── 概览 / OVERVIEW                    [新] 仪表盘首页
│   ├── 今日新增观点
│   ├── 待处理队列
│   ├── KOL 评分排行
│   └── 市场情绪指标
│
├── 观点追踪 / OPINIONS                [新] 核心功能
│   ├── 全部观点
│   │   ├── 按标的筛选
│   │   ├── 按 KOL 筛选
│   │   └── 按状态筛选（待验证/已验证/失效）
│   ├── KOL 评价 → 详情见第三章
│   └── 观点时间线
│
├── 素材库 / ASSETS                    [合并 L0-L2]
│   ├── 接入台（新导入）
│   ├── 富化台（话题拆分）
│   └── 知识库（已归档）
│
├── 处理工坊 / WORKSHOP                [合并 L3-L6]
│   ├── 解析结果
│   ├── 候选事件
│   └── 复核队列
│
├── 验证中心 / VALIDATION              [L8 升级]
│   ├── 回测结果
│   ├── 收益统计
│   └── 策略评估
│
└── 集成中心 / INTEGRATIONS
    ├── 飞书同步
    └── NotebookLM
```

### 导航层级优化

**Before（7 层平铺）**：
```
L0 → L1 → L2 → L3 → L5 → L6 → L8
```

**After（3 层分类）**：
```
概览 → 观点追踪 → 素材库 → 处理工坊 → 验证中心
         ↓
    [KOL 评价详情]
```

### 快捷路径设计

| 用户目标 | 原路径 | 新路径 | 步骤减少 |
|---------|--------|--------|---------|
| 追踪某 KOL 观点 | 无直接入口 | 观点追踪 → KOL 评价 → 选择 KOL | 新增功能 |
| 查看待处理观点 | L5 → 逐个查看 | 概览 → 待处理队列 | -2 步 |
| 导入新素材 | L0 → 导入 | 素材库 → 接入台 → 导入 | 持平 |
| 验证观点准确性 | L6 → L8 | 验证中心 → 选择观点 | -1 步 |

## 三、KOL 评价系统 UX 设计

### 设计参考

参考晨星网基金经理评价体系，提取核心设计元素：

| 设计元素 | 晨星实践 | KOL 评价应用 |
|---------|---------|-------------|
| 星级评级 | 1-5 星基金经理 | 1-5 星 KOL |
| 时间线 | 历史业绩曲线 | 观点准确率时间线 |
| 仪表盘 | 风险/收益仪表 | 准确率/止损/一致性仪表 |
| 数据卡片 | 持仓详情 | 专注领域分布 |
| 展开/折叠 | 点击查看详情 | 点击展开 KOL 详情 |
| 时间范围 | 1年/3年/5年 | 1月/3月/6月/1年 |

### 评价维度设计

#### 核心指标

```
KOL 评价维度
├── 观点准确率 (Weight: 35%)
│   ├── 多头观点准确率
│   ├── 空头观点准确率
│   └── 中性观点准确率
│
├── 止损质量 (Weight: 25%)
│   ├── 止损及时性
│   ├── 止损幅度合理性
│   └── 止损后复盘质量
│
├── 观点一致性 (Weight: 20%)
│   ├── 逻辑连贯性
│   ├── 前后观点不矛盾
│   └── 立场稳定性
│
├── 市场敏感度 (Weight: 10%)
│   ├── 观点时效性
│   ├── 市场转折点捕捉
│   └── 信息来源多样性
│
└── 风险提示 (Weight: 10%)
    ├── 风险提及频率
    ├── 风险情景覆盖
    └── 止损建议明确度
```

#### 星级评定规则

| 星级 | 综合得分 | 含义 | 可视化 |
|-----|---------|------|--------|
| 5 星 | ≥ 90 | 卓越 | 金奖章 |
| 4 星 | 75-89 | 优秀 | 银奖章 |
| 3 星 | 60-74 | 良好 | 铜奖章 |
| 2 星 | 40-59 | 一般 | 无奖章 |
| 1 星 | < 40 | 较差 | 无奖章 |

### 交互规范

#### 1. 展开/折叠

**默认状态**：显示顶部摘要区 + 中部核心指标

**展开触发**：
- 点击卡片任意位置
- 点击「查看详情」按钮
- 键盘 Enter 键

**展开后显示**：
- 完整中部详情区
- 底部历史区

#### 2. 时间范围切换

**切换选项**：1月 | 3月 | 6月 | 1年 | 全部

**切换效果**：
- 时间线图表更新
- 准确率数据重算
- 观点列表筛选

**默认选择**：3月

#### 3. 观点详情跳转

**触发方式**：点击「查看详情」

**跳转目标**：观点详情页（显示完整证据链）

#### 4. 关注/取消关注

**关注状态**：
- 未关注：灰色按钮「添加关注」
- 已关注：红色按钮「已关注」

**状态切换**：点击按钮切换

### 晨星设计参考总结

| 晨星设计元素 | 应用到 KOL 评价 |
|-------------|----------------|
| 星级评级 | 1-5 星 KOL 评级 + 金银铜奖章 |
| 仪表盘 | 准确率/止损/一致性三仪表 |
| 时间线 | 观点验证时间线 |
| 数据卡片 | 专注领域分布 |
| 展开/折叠 | 卡片点击展开详情 |
| 时间范围切换 | 1月/3月/6月/1年 切换 |
| 摘要区 | 顶部 KOL 核心信息 |
| 详情区 | 中部分维度指标 |
| 历史区 | 底部时间线 + 最近观点 |

### 设计决策记录

| 决策 | 选择 | 原因 |
|-----|------|-----|
| 星级数量 | 5 星 | 与晨星保持一致，用户熟悉 |
| 评价指标数量 | 5 个 | 平衡全面性与简洁性 |
| 时间线默认范围 | 3 月 | 平衡数据量与趋势可见性 |
| 卡片展开方式 | 点击展开 | 减少初始信息密度 |
| 导航层级 | 3 层 | 减少认知负担 |
