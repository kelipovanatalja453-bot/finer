🔄 六层流水线详解

📥

L0 接入台

接收来自飞书、微信公众号、B站等多源的原始素材，自动分类归档。

# 支持的数据源 sources = [ "feishu", # 飞书群消息 "wechat", # 微信公众号 "bilibili", # B站视频转录 "notebooklm", # NotebookLM 同步 ]
🏷️

L1 富化台

话题拆分、实体抽取、内容关联，构建结构化知识图谱。

# 富化输出示例 { "entities": ["腾讯", "美团"], "topics": ["港股", "科技股"], "sentiment": "bullish" }
🔍

L3 解析台

OCR 文字识别、视觉转录、音频转文字，提取结构化内容。

⚡

L5 抽取台

GLM-5.1 提取 TradeAction，包含标的、方向、操作链、置信度。

# TradeAction 输出 { "ticker": "0700.HK", "direction": "bullish", "action_chain": [ {"action": "watch", "at": 480}, {"action": "long", "on": "breakout"} ] }
✅

L6 复核台

RLHF 人工评价，收集反馈数据用于模型微调。

📈

L8 回测台

验证观点准确性，计算 KOL 评级，生成回测报告。

✨ 核心功能

🔌

多源数据导入

支持飞书群消息、微信公众号文章、B站视频转录等多种数据源，统一接入 L0 层处理。

📸 数据源配置界面截图
🤖

智能 Trade Action 提取

使用 GLM-5.1 + Finance-Skills 混合策略，从非结构化文本提取结构化交易观点。

# 输入：KOL 观点文本 text = "腾讯日线支撑480，破了就止损，突破看520" # 输出：结构化 TradeAction { "ticker": "0700.HK", "direction": "bullish", "confidence": 0.85, "action_chain": [ {"seq": 1, "action": "watch", "trigger": "price <= 480"}, {"seq": 2, "action": "long", "trigger": "breakout > 500"}, {"seq": 3, "action": "stop_loss", "trigger": "price < 480"} ] }

👥 RLHF 评价系统

为什么需要人工评价？
模型可能错误识别标的、方向判断失误、操作链逻辑混乱。人工反馈用于：

  • 修正模型错误输出
  • 收集 DPO 训练数据
  • 持续提升模型准确率

🏆 KOL 评价体系

参考晨星网基金经理评价体系，为 KOL 建立多维度评分：

维度 权重 说明
观点准确率 35% 看多/看空/中性各自的准确率
止损质量 25% 止损建议的及时性和合理性
一致性 20% 观点逻辑连贯性
市场敏感度 10% 观点领先性
风险提示 10% 风险提示频率和覆盖
